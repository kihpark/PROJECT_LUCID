# Lucid — 프로젝트 컨텍스트 브리핑 (Claude / NotebookLM 공유용)

> 이 문서는 Lucid 벤처에 대한 모든 핵심 맥락을 담은 단일 브리핑이다.
> 새 Claude 대화 시작 시 붙여넣거나, NotebookLM 소스로 업로드해 메모리/맥락을 공유하는 용도다.
> 작성일 기준: 2026-05-18.

## 0. 이 문서를 받은 AI에게 (사용 지침)

당신은 지금부터 **Lucid**라는 벤처 프로젝트의 맥락을 인계받는다. 아래 사항을 지켜라.

- Lucid는 노트 앱이 아니라 **검증 인프라(validation infrastructure)** 다. 이 프레이밍을 절대 흐리지 마라.
- 한국어로 응답할 때는 **영어를 직역한 어색한 말투를 쓰지 말 것.** 한국어로 네이티브하게 쓰고, 서정적 묘사보다 **논리적 전제-구조**를 앞세워라.
- 창업자는 날카로운 전략적 사고를 한다. 출처/인용의 정확성, 서사의 논리적 일관성에 민감하다. 추측을 사실처럼 말하지 마라.
- 브랜드명·슬로건·태그라인·자금 전략은 **확정(locked)** 사항이다. 임의로 바꾸지 마라.

## 1. 한 줄 정의

**Lucid = "검증된 두 번째 뇌(verified second brain)".**
사용자가 마찰 없이 캡처한 정보를 AI가 원자적 사실로 구조화하고, 인간이 검증(HITL)하며, 검증을 통과한 지식만 개인 지식 그래프에 영구 저장된다. 모든 사실은 **4단계 검증 서명**(L1 본인 / L2 신뢰망 / L3 시스템 / L4 전문가)을 보유한다.

**핵심 명제:** AI가 콘텐츠를 무한히 생성하는 시대에, 단 하나 희소해지는 자원은 **검증된 진실**이다. 그리고 검증은 인간의 시간을 필요로 하므로 복제할 수 없다.

## 3. 출신 — 두 MVP의 통합

Lucid는 창업자가 이미 만든 **두 개의 풀스택 MVP**의 메커니즘을 통합·발전시킨 결과물이다.

1. **Student** — 사실 추출·검증·질의응답 엔진. CMU의 *"Student: an Intelligent Machine that Learns from People"* (Gershman, 2019) 제안서를 구현. `student-demo/` 폴더에 존재 (FastAPI + SQLite + Anthropic SDK 백엔드, React + Vite 프론트엔드). 핵심 불변식: Student는 DB에 저장된 사실만으로 답하고 LLM 일반지식으로 답하지 않는다.
2. **WisdomDB** — 캡처 + HITL + Neo4j 그래프. 창업자의 별도 선행 프로젝트. 브라우저 확장·모바일 PWA·멀티모달 추출기·Neo4j 그래프·HITL 워크플로우 보유.

> 창업자는 외부 자금 없이 6개월 만에 단독으로 이 두 풀스택 제품을 완성했다. Lucid 통합 아키텍처 설계는 이미 끝났고 8주 안에 베타가 가능하다.

**중요:** Lucid는 `student-demo/`와 **별개의 벤처 프로젝트**다. 폴더도 의도적으로 분리됨 — `Lucid/`.

## 4. 문제 (Problem)

두 위기가 겹쳤다.

1. **기억의 위기** — 매일 평균 34GB 정보 노출(UCSD 2009)이지만 재사용 가능한 형태로 남는 건 5% 미만. 지식근로자는 주당 9.3시간을 "이미 본 정보를 다시 찾는 데" 소비(McKinsey).
2. **신뢰의 위기** — 2024년 기점, AI 생성 콘텐츠량이 인간 작성량을 추월(Europol). 문제는 "정보를 찾는 일"이 아니라 "찾은 정보를 믿어도 되는지 판단하는 일"이 불가능해진 것.

두 위기의 뿌리는 같다: 인간 인지 체계(작업기억 ~7항목)는 이 규모·불확실성을 감당하도록 설계되지 않았다.

## 5. 통찰 (The Insight)

기존 해법들의 공통 맹점:
- **검색 엔진** — "어디 있는가"만 풀고 "믿을 수 있는가"는 안 푼다.
- **노트 앱** (Notion/Obsidian/Evernote) — 저장된 모든 것을 동등한 진실로 취급. 분류도 수작업.
- **AI 어시스턴트** (ChatGPT) — "평균적 지식"으로 그럴듯하게 틀리고, 내 검증된 지식이 아니다.

**핵심 통찰:** 정보 과부하의 진짜 병목은 저장 용량이 아니라 **검증**이다. 그리고 검증은 자동화할 수 없다 — "이것이 참인가"의 최종 판단 책임은 인간에게 있다. 다음 세대 지식 도구는 **AI의 생성 능력과 인간의 판단 능력을 분업**시켜야 한다. AI가 초안, 인간이 판정.

## 6. 제품 — 4단계 루프

| 단계 | 무엇이 일어나는가 | 누가 | 마찰 |
|---|---|---|---|
| **캡처 (Capture)** | 웹·모바일·멀티모달 정보를 그 자리에서 저장 | 사용자 + AI | < 5초 |
| **구조화 (Structure)** | AI가 원자적 사실·개체·관계로 정리 | AI | 자동 |
| **검증 (Validate)** | 수락 / 편집 / 폐기로 진위 판정 | 사용자 (HITL) | < 30초 |
| **소환 (Surface)** | 필요한 맥락에서 검증된 사실이 자동으로 떠오름 | AI | 사용자 무자각 |

예시 시나리오: 기후정책 연구원이 유튜브 강연을 보다 우클릭 → "Lucid에 저장"(5초). AI가 3개 사실 주장 추출 → 사용자가 30초 안에 수락/불확실/폐기 판정. 수락된 사실은 개인 지식 그래프에 편입돼 기존 사실과 자동 연결. 3주 후 보고서 작성 중 "재생에너지" 입력 시 사이드바에 그 사실이 자동 소환 + 출처 자동 인용.

## 7. 검증 시스템 — 4단계 (핵심 차별점)

모든 사실은 "저장됨/안 됨" 이분법이 아니라 **네 겹의 검증 표식**을 동시에 보유한다.

| 층위 | 검증 주체 | 표시 예 | 구현 우선순위 | 대응 수익 라인 |
|---|---|---|---|---|
| **L1 본인** | 사용자 본인 (HITL) | 직접 수락, 출처 N개 교차확인 | M0 — 이미 완성 | Pro 구독 |
| **L2 신뢰망** | 친구·팀·구독 사용자 | "신뢰망 47명 중 38명 동의(81%)" | M12 | Team tier |
| **L3 시스템** | 전체 사용자 익명 집계 | "전체 6,421명 중 73% 동의" | M6 — 사용자 모이면 자동 | 플랫폼(네트워크 효과) |
| **L4 전문가** | 도메인 전문가 인증 | "에너지경제 박사 OOO 검증함" | M18+ | Marketplace / Enterprise |

추가로: **반대 의견 지도**("19%는 상반된 사실 저장 중") + **출처 추적**(N단계 거슬러 추적). 4개 층위는 동시에 **단계적 시장 진입 경로**이기도 하다.

## 8. 방어 가능성 (Defensibility)

투자자의 단골 질문: "OpenAI가 다음 버전에서 이 기능을 추가하면?"

답: Lucid가 누적하는 자산은 모델 가중치도 인터페이스도 아닌, **수천·수만 사용자가 각자 30초씩 들인 검증 판단의 총합**이다. 더 큰 모델·더 긴 컨텍스트는 콘텐츠 생성을 가속하지만 "이것이 참인가" 판단의 인간 시간은 단축하지 못한다.

복리로 겹치는 4개 해자:
1. **전환 비용** — 떠나면 수년간 검증한 인지 자산 전체를 버려야 함
2. **개인화 복리** — 쓸수록 그 사용자 사고 패턴에 맞춰 소환 정확도 상승
3. **검증 데이터 희소성** — 인간 검증 사실 그래프는 AI 회사가 못 가진 학습 데이터
4. **네트워크 효과** — L3는 사용자 늘수록 정밀해짐 (1천명 vs 100만명은 질적으로 다름)

> "12개월 선행만 확보하면, 추격은 자본이 아니라 시간의 문제가 된다."

미리 해소한 반론 3가지: ① Notion/Obsidian의 또 다른 버전 아닌가? → 그들은 저장 도구, Lucid는 검증 카테고리. ② 검증에 30초 쓸 사람 있나? → 초기 타깃은 진위에 직업적으로 민감한 사람들. ③ AI가 검증까지 하면? → 최종 판단을 AI에 맡기면 다시 "그럴듯하게 틀린 답"의 세계.

## 10. 경쟁 (Competition)

**주요 경쟁자는 Obsidian** (Notion이 아님). Obsidian: 직원 ~15명, 추정 연매출 $30M+, 영업이익률 80%+, VC 자금 0, 로컬 마크다운 저장, 1,500+ 플러그인.

그러나 Obsidian의 강점 구조(로컬 우선·클라우드 거부·자체자본)가 곧 한계 — **AI-네이티브가 될 수 없다.** Obsidian AI 기능은 외부 플러그인 의존.

**Lucid의 타깃:** Obsidian 헤비유저를 빼앗지 않는다. ① Obsidian 시도했다 복잡함에 이탈한 사람들(수백만) ② Notion 1억 명과 Obsidian 7백만 명 사이 간극의 9천만+ 명.

2×2 포지셔닝:
| | 일반 데이터(인터넷) | 개인 데이터(내 것) |
|---|---|---|
| **검증/큐레이션됨** | Wikipedia·Snopes | ★ **Lucid (빈 분면)** |
| **원본/비검증** | Google·Perplexity·ChatGPT | Obsidian·Notion·Mem·Rewind |

## 11. 비즈니스 모델

| Tier | 가격 | 검증 층위 | 타깃 |
|---|---|---|---|
| Free | $0 | L1 + L3 일부 | 유입·체험 |
| Pro | $19/월 | L1 + L3 + L2 미리보기 | 개인 지식근로자 |
| Team | $39/seat | L1 + L2 + L3 | 스터디·연구실·소규모 팀 |
| Enterprise | $50~200/seat/월 | L1~L4 + 전문가 풀 | 법무·R&D·컨설팅·미디어 |
| Validation API | $10K~50K/년 | L3 + provenance API | AI 회사·검색엔진·출판사 |

**Validation API**는 새 영역 — AI 회사가 자사 출력을 Lucid 검증 그래프와 교차확인. 잠재 고객: Anthropic·OpenAI·Perplexity.

**유닛 이코노믹스 (Pro):** LLM 추론 ~$3/월/사용자(Sonnet, prompt caching) → gross margin ~78%. 12개월 retention 65% 추정. LTV ~$310, 목표 CAC $90 → LTV/CAC 3.4배. CAC 회수 ~5개월.

## 12. 자금 전략 (확정) — 단계적 접근

표준 $1.5M SV 시드 라운드를 **택하지 않는다.** 검증 인프라는 빠른 성장이 아니라 깊은 축적이 가치인 비즈니스.

- **Phase 0 — 부트스트랩 + 엔젤** (0~6개월, 현재 라운드): 5천만~1.5억 원. 솔로 + 파트타임 엔지니어 1명. 6개월 비용 ~$36,650.
- **Phase 1 — 한국 시드** (6~18개월): 3~10억 원. 대상 — Sparklabs, Primer, Mashup Angels, Strong Ventures. 창업자 풀타임 전환, 창립 엔지니어·디자이너 채용.
- **Phase 2 — 시리즈 A 또는 흑자** (18개월+): ARR $1M + NRR 110%↑면 시리즈 A, 아니면 ARR $2M 흑자로 자체 운영(Obsidian 모델).

**결정 게이트:** Phase 0가 **페잉 사용자 50명 + retention 60%↑ + NPS 40↑** 로 끝날 때에만 Phase 1 진입. 미달 시 가설 재검토.

**3년 궤적:** M6 유료 50명/$12K ARR → M12 400명/$96K → M18 1,500명/$360K → M24 5,000명/$1.2M → M36 18,000명/$4.3M ARR.

## 13. 창업자

CMU AI 계보 출신. Lucid의 지적 뿌리인 SCHOLAR(Carbonell, 1970)와 Student 제안서(2019)가 모두 CMU에서 나옴. 외부 자금 없이 6개월 만에 단독으로 두 풀스택 제품(WisdomDB, Student) 출시. 투자는 "0→1"이 아니라 "검증된 1→10"에 쓰인다.

채용 계획: Phase 1에 창립 엔지니어(벡터·그래프 검색)·프로덕트 디자이너, 후반에 GTM 리드. 자문단 목표: CMU LTI 교수, Wikipedia/Glean 출신, 한국 EdTech 시리얼 창업자.

## 15. 비전 & 지적 계보

**비전:** 한 세대 뒤, 사람들은 이메일·캘린더를 갖듯 자신의 검증된 지식 그래프를 갖는다. 개인 그래프들이 익명으로 합쳐져 인류가 — 광고·알고리즘이 아니라 — 검증된 진실의 지도를 공유한다.
