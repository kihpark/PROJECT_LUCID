How it works

작동 방식

  • User types in any web text field (Gmail, Google Docs, Notion, Slack, etc.) via Chrome extension
  • 500ms debounce after keystroke — Lucid scans keywords against the graph
  • Matching phrases get a dotted orange underline (subtle, non-intrusive)
  • Hover the underline → tooltip appears with top 3 related facts
  • "Insert citation" puts fn-ID inline at cursor; "See all" opens the See All panel (S-2)
  • 사용자가 어느 웹 텍스트 필드든 입력 (Gmail, Google Docs, Notion, Slack 등 Chrome 확장)
  • 키 입력 후 500ms 디바운스 — Lucid가 그래프 대조
  • 일치 구문에 점선 주황 밑줄 (은은함, 비방해)
  • 밑줄 호버 → 풍선에 상위 3개 관련 사실
  • "인용 삽입"은 fn-ID를 커서 위치에; "전체 7개"는 See All 패널(S-2) 오픈

Design choices

디자인 결정

  • Dotted underline (not solid) — signals "informational" not "error"
  • Orange (not red) — distinct from spell-check or grammar tools
  • Tooltip appears on hover only — no popup while typing
  • Lucid ON/OFF toggle (extension popup) instantly disables underlines
  • 점선 밑줄 (실선 아님) — "정보용" 신호, "오류"가 아님
  • 주황색 (빨강 아님) — 맞춤법·문법 도구와 구분
  • 풍선은 호버 시에만 — 타이핑 중 팝업 없음
  • 확장 팝업의 Lucid ON/OFF 토글로 즉시 밑줄 비활성화

Identity protocol — every response signed

정체성 규약 — 모든 응답에 서명

Every Lucid response begins with one of these phrases:

모든 Lucid 응답은 다음 표현 중 하나로 시작:

  • "As far as I know,"
  • "According to your knowledge graph,"
  • "Based on what you've validated,"
  • "제가 아는 한,"
  • "그래프 기준으로,"
  • "기흥님이 검증한 것 기준으로,"

This is not stylistic. It's an epistemic commitment: Lucid never speaks beyond what the user has validated. No "I think...", no general knowledge fill-ins, no plausible-sounding inventions.

단순 말투가 아닙니다. 인식론적 약속: Lucid는 사용자가 검증한 것을 넘어 말하지 않습니다. "내 생각엔...", 일반 지식 메우기, 그럴듯한 창작 없음.

Relationship tree — understanding is structure, not list

관계 트리 — 이해는 구조, 목록이 아니다

After listing facts, Lucid surfaces the relationships between them (SUPPORTS, SUPERSEDES, CONTRADICTS, DERIVED_FROM). This is intentional: understanding is not a list of facts, it's the structure of how facts relate. The relationship tree makes the user feel they are "inside the loop of understanding" with the AI, not just receiving a summary.

사실 나열 후 Lucid는 사실들 사이의 관계 를 보여줌 (SUPPORTS, SUPERSEDES, CONTRADICTS, DERIVED_FROM). 의도적: 이해는 사실의 목록이 아니라 사실들이 어떻게 연결되는지의 구조. 관계 트리는 사용자가 단순 요약을 받는 게 아니라 AI 와 "이해의 루프 안에" 함께 있다는 느낌을 만듦.

Without the tree, Lucid is a smart search. With the tree, Lucid surfaces the user's own coherent narrative back to them. This is the bridge from "verified knowledge" to "structured understanding."

관계 트리가 없으면 Lucid는 똑똑한 검색에 불과. 있으면 사용자 자신의 일관된 narrative 를 다시 보여주는 시스템. "검증된 지식" 에서 "구조화된 이해" 로 가는 다리.

Three response patterns

세 가지 응답 패턴

  • Affirmative — Graph has matching facts. List them with fn-IDs. (Example 1: protein timing)
  • Contradictive — User's claim conflicts with graph. Show conflict, offer to save the new claim for review. (Example 2: fasted cardio)
  • Honest empty — Graph has nothing. Don't invent. Suggest saving. (Example 3: creatine)
  • 긍정 — 그래프에 일치 사실 있음. fn-ID와 함께 나열. (예시 1: 단백질 타이밍)
  • 반박 — 사용자 주장이 그래프와 충돌. 충돌 보여주고 검토 큐 저장 제안. (예시 2: 공복 유산소)
  • 정직한 빈 답변 — 그래프에 없음. 창작 안 함. 저장 제안. (예시 3: 크레아틴)

Why this is the killer feature

왜 이게 킬러 기능인가

ChatGPT and Claude answer from training data — a vast averaged sea. Lucid answers only from what you've personally validated. Same question, very different answer: "What do I know about X?" — only Lucid can answer that question literally and meaningfully.

ChatGPT나 Claude는 학습 데이터 — 거대한 평균의 바다 — 에서 답합니다. Lucid는 사용자가 직접 검증한 것에서만 답합니다. 같은 질문 다른 의미: "내가 X에 대해 뭘 알지?" — 이 질문에 문자 그대로 의미 있게 답할 수 있는 건 Lucid뿐입니다.

How contradictions are detected

모순 감지 방식

  • Pattern A — Same Subject + same Property + different value, same time/context. Auto-flagged.
  • Pattern B — Same Subject + semantically opposite Predicates. Suggested for review.
  • Pattern C — Same Subject + Property but different time/jurisdiction. NOT a contradiction — both preserved (timeline).
  • 패턴 A — 같은 대상 + 같은 속성 + 다른 값, 같은 시점/맥락. 자동 플래그.
  • 패턴 B — 같은 대상 + 의미적으로 반대되는 술어. 검토 제안.
  • 패턴 C — 같은 대상 + 속성이지만 다른 시점/관할. 모순 아님 — 둘 다 보존 (타임라인).

When detection runs

언제 감지가 작동하나

  • Immediately after Accept (synchronous quick check)
  • Background daily scan (full graph sweep)
  • Whenever a new fact connects to an existing Subject node
  • Accept 직후 (동기 빠른 체크)
  • 매일 백그라운드 스캔 (전체 그래프)
  • 새 사실이 기존 Subject 노드에 연결될 때마다

Four resolution options

4가지 해소 옵션

  • Keep one, drop the other — Clear choice when one source is clearly stronger
  • Keep both with context note — When difference is due to jurisdiction, scope, or interpretation
  • Demote both — Send back to Pending for fresh re-review
  • 하나 유지, 다른 것 폐기 — 한쪽 출처가 명확히 강할 때
  • 맥락 메모와 함께 둘 다 유지 — 차이가 관할/범위/해석 때문일 때
  • 둘 다 강등 — 다시 신선하게 재검토

No intrusive toast alerts

방해적 토스트 알림 없음

Contradictions are surfaced via sidebar badge (Insights → Contradictions) and visual signal in Stellar View (red tension lines), but never as a popup interrupting the user's flow.

모순은 사이드바 배지 (Insights → Contradictions)와 Stellar View 시각 신호 (붉은 긴장선)로 표시되며, 사용자 흐름을 끊는 팝업은 절대 없습니다.

When does Gatekeeping warn?

언제 Gatekeeping이 경고하나

Only when ALL three conditions are met:

다음 세 조건이 모두 충족될 때만:

  • Multiple existing facts (≥2) contradict the new claim
  • The new claim's source is weaker than counter-evidence sources
  • Counter-evidence is at least as recent as the new claim
  • 여러 기존 사실 (2건 이상)이 새 주장과 충돌
  • 새 주장의 출처가 반대 증거 출처보다 약함
  • 반대 증거가 새 주장만큼 또는 더 최근

Conservative threshold — most contradictions go to the Contradiction Queue (S-4) for normal review. Gatekeeping only triggers on cases that look like potential misinformation or strongly debunked claims.

보수적 임계값 — 대부분의 모순은 Contradiction Queue (S-4)로 가서 일반 검토. Gatekeeping은 가능한 가짜 정보나 강하게 반박된 주장 케이스에만 발동.

Warn, never block

경고만, 차단 없음

Lucid does not block saves. The user can always override. If they do, the saved fact is marked override_warning: true in metadata, and is visually distinct in the graph (yellow border). This preserves user autonomy while maintaining transparency.

Lucid는 저장을 차단하지 않습니다. 사용자는 언제든 강제 저장 가능. 강제 저장한 사실은 메타데이터에 override_warning: true로 표시되고, 그래프에서 시각적으로 구분됨 (노란 테두리). 사용자 자율성을 유지하면서 투명성 보장.

Beta scope — self-graph only

베타 범위 — 자기 그래프만

In beta, Gatekeeping compares only against the user's own validated facts. Phase 1 will add comparison against followed experts' graphs. Phase 2 may add external fact-check databases. This keeps the user in control of what counts as authoritative.

베타에서는 사용자 자신의 검증된 사실만 비교. Phase 1에서는 팔로우한 전문가의 그래프 비교 추가. Phase 2에서는 외부 팩트체크 DB 추가 가능. 사용자가 권위의 기준을 통제하도록 유지.
