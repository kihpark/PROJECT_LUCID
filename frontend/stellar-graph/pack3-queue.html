How items arrive here

여기로 들어오는 경로

  • User pressed ESC on the Decide overlay during save (deferred decision)
  • User closed the overlay without choosing Accept/Review/Discard
  • Auto-accepted facts that triggered contradictions (re-entered as a group for review)
  • 저장 시 Decide 오버레이에서 ESC를 누름 (결정 미루기)
  • Accept/Review/Discard 선택 없이 오버레이를 닫음
  • 모순이 감지된 자동 수락 사실들 (그룹으로 재진입)

Visual signals

시각 신호

  • Red left border — contradiction detected with existing graph facts
  • Grey left border — capture is getting old (3+ days)
  • Warn pill — count of facts that conflict with what's already in the graph
  • 붉은 좌측 테두리 — 기존 그래프 사실과 모순 감지
  • 회색 좌측 테두리 — 캡처가 오래됨 (3일 이상)
  • 경고 pill — 그래프 기존 사실과 충돌하는 사실 수

Sort and filter

정렬과 필터

  • Recent — most recent capture first (default)
  • By source — group together captures from same source
  • By size — captures with most facts first (clear those in one pass)
  • Search filters by title, source, or fact content
  • 최근 — 최근 캡처 먼저 (기본)
  • 출처별 — 같은 출처 캡처를 묶음
  • 크기별 — 사실 수 많은 캡처 먼저 (한 번에 많이 처리)
  • 검색은 제목, 출처, 사실 내용으로 필터링

Pack 2 C-3와 동일한 UI — 의도된 설계

The Decide overlay is the same component whether opened on the source page (Pack 2) or from the queue (here). Same three options, same data. The only differences:

Decide 오버레이는 출처 페이지에서 열리든(Pack 2) 큐에서 열리든(여기) 같은 컴포넌트. 같은 3가지 옵션, 같은 데이터. 차이는:

Why contradictions get detected even before Accept

왜 Accept 전에도 모순이 감지되는가

Lucid compares pending facts against the graph as soon as analysis completes. This protects the graph: if a deferred capture sat for 3 weeks and the graph evolved, the user sees fresh contradiction warnings when they finally come back. The fn-187 reference lets the user click through to the conflicting fact and decide which is right.

Lucid는 분석 완료 직후 그래프와 pending 사실들을 대조함. 그래프 보호를 위해: 캡처를 3주간 미뤄두고 그동안 그래프가 진화했다면, 사용자가 돌아왔을 때 새로운 모순 경고를 봄. fn-187 참조를 클릭하면 충돌 사실로 이동해서 어느 쪽이 맞는지 결정 가능.

What "auto-accepted" means

"자동 수락"의 의미

These facts entered the graph without the user reviewing each one — because the source was registered as trusted in Settings. Each fact carries an L1 auto mark indicating no manual validation, but trusted source provenance. The user retains full control to edit, demote, or drop any of them.

이 사실들은 사용자가 하나씩 검토하지 않고 그래프에 진입했다 — 출처가 Settings에서 신뢰 등록되어 있었기 때문. 각 사실에는 L1 auto 마크가 붙어, 수동 검증은 없지만 신뢰 출처에서 왔음을 표시. 사용자는 언제든 수정·강등·폐기 가능.

Three actions per fact

사실당 3가지 액션

  • Edit — Inline edit the claim or tags. Alias history preserved.
  • Demote — Move fact back to Pending queue for explicit review. Useful when you want to double-check before relying on it.
  • Drop — Remove fact from graph entirely. Cannot be undone (only recoverable via re-capture from source).
  • 수정 — 주장 또는 태그를 인라인 편집. Alias 이력 보존.
  • 강등 — 사실을 Pending 큐로 되돌려 명시적 검토. 신뢰하기 전에 한 번 더 확인하고 싶을 때.
  • 폐기 — 그래프에서 완전 제거. 되돌릴 수 없음 (출처에서 다시 캡처해야 복구 가능).

Time-stamped facts are permanent

시점 사실은 영구 보존

Lucid does not expire facts. A fact like "BoK base rate was 3.5% in December 2024" is true forever — the time context is part of the fact itself. When new facts arrive ("3.0% in March 2025"), both stay in the graph; together they form the timeline. Surface answers prioritize the most recent matching fact when relevant.

Lucid는 사실을 만료시키지 않습니다. "2024년 12월 기준 한국 금리 3.5%" 같은 사실은 영원히 진실 — 시점 컨텍스트가 사실의 일부. 새 사실("2025년 3월 3.0%")이 들어오면 양쪽 모두 그래프에 보존되어 함께 타임라인을 형성. Surface 응답은 관련된 경우 최신 일치 사실을 우선 표시.

Why audit auto-accepted at all?

왜 자동 수락된 사실을 감사하는가

Trusted means "I trust this source's editorial process" — not "everything from this source is correct." Sources publish corrections, change positions, get reanalyzed. Auto-accepted tab gives the user a single audit surface to catch issues without manually reviewing every save.

신뢰는 "이 출처의 편집 절차를 신뢰한다"는 뜻 — "이 출처에서 나온 모든 것이 맞다"는 뜻이 아님. 출처는 정정 발행, 입장 변경, 재분석될 수 있음. 자동 수락 탭은 매번 수동 검토 없이도 문제를 잡을 수 있는 감사 화면.
