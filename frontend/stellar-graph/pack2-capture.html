What "Save to Lucid" means

"Save to Lucid"의 의미

This click hands the page to Lucid for analysis. For trusted sources, facts enter the graph immediately. For other sources, the user sees a Decide overlay (Step 3) and chooses what to do with the extracted facts.

이 클릭은 페이지를 Lucid에게 분석 맡기는 동작. 신뢰 등록된 출처면 즉시 그래프 진입. 그 외 출처는 사용자가 Decide 오버레이(3단계)에서 추출 결과를 보고 결정.

Notification model — toast during, badge after

알림 모델 — 진행 중 toast, 완료 후 badge

  • Toast shown during analysis (3–10 sec typical). Non-blocking, dismissable. User can keep reading.
  • Badge on extension icon — "…" during analysis, switches to a number when ready (e.g., "3").
  • On completion, toast briefly shows a green check then disappears. Badge persists until user opens Decide overlay (C-3).
  • If user navigated away, badge alone signals completion. They return to Decide anytime via extension popup.
  • Toast는 분석 중 표시 (보통 3–10초). 방해 없음, 닫기 가능. 사용자는 계속 읽을 수 있음.
  • Badge는 확장 아이콘에 표시 — 분석 중 "…", 완료 시 숫자 ("3").
  • 완료 시 toast가 잠시 녹색 체크를 보인 후 사라짐. Badge는 Decide 오버레이(C-3)를 열 때까지 유지.
  • 사용자가 다른 곳으로 갔어도 badge가 완료를 알림. 확장 팝업에서 언제든 Decide로 복귀 가능.

Trusted source — automatic flow

신뢰 출처 — 자동 흐름

If the source is registered as trusted, Step 3 is skipped entirely. Toast shows "✓ Saved to graph · 3 facts added" then disappears. Facts enter the graph with L1 auto-accept marks. Reviewable later in the Auto-accepted tab.

출처가 신뢰 등록되어 있으면 3단계 완전 생략. Toast가 "✓ 그래프에 저장됨 · 사실 3개 추가" 표시 후 사라짐. 사실들은 L1 auto-accept 마크와 함께 그래프 진입. Auto-accepted 탭에서 사후 검토 가능.

Three decisions, mapped to user intent

세 가지 결정, 사용자 의도 매핑

  • Accept all — Most common path. All facts enter graph as L1.
  • Review one-by-one — Want to inspect each fact. Overlay switches to Review mode (C-4).
  • Discard all — Extraction is wrong, or changed mind. Complete discard. To save again, start the capture flow over.
  • 모두 승인 — 가장 일반적 경로. 모든 사실이 L1으로 그래프 진입.
  • 개별 검토 — 사실 하나씩 검토. 오버레이가 Review 모드(C-4)로 전환.
  • 모두 폐기 — 추출이 잘못됐거나 마음이 바뀌었을 때. 완전 폐기. 다시 저장하려면 캡처 흐름 처음부터.

ESC = defer, not decide

ESC = 미루기, 결정 아님

Closing the overlay without choosing is treated as "defer". Facts stay in Pending Queue, accessible later from extension popup or the Lucid app. This protects users who get interrupted mid-decision.

선택 없이 오버레이 닫기 = "미루기". 사실은 Pending Queue에 남아 나중에 확장 팝업 또는 Lucid 앱에서 접근 가능. 결정 도중 방해받는 사용자를 보호.

Why Review mode lives in the overlay

Review 모드가 오버레이 안에 있는 이유

Review is a continuation of the same decision moment. User is still on the source page; context is still fresh. Forcing navigation to a separate validation screen would break flow. The overlay simply switches modes.

Review는 같은 결정 순간의 연장. 사용자는 출처 페이지에 그대로 있고 맥락도 신선. 별도 검증 화면으로 이동하면 흐름이 끊김. 오버레이는 모드만 전환.

Three actions per fact

사실당 3가지 행동

  • Accept — Fact enters graph with L1 mark. Next card appears.
  • Edit — Inline edit field opens. Original preserved as alias. Save → enter graph.
  • Reject — Fact discarded. Next card appears.
  • 승인 — 사실이 L1 마크와 함께 그래프 진입. 다음 카드 표시.
  • 수정 — 인라인 편집 필드 열림. 원본은 alias로 보존. 저장 → 그래프 진입.
  • 폐기 — 사실 폐기. 다음 카드 표시.

Escape hatches

중간에 빠져나오기

Bottom hint shows "Accept rest" and "Discard rest" — finish the batch in one click after handling a few. "Back to summary" returns to C-3 without losing progress on already-decided facts.

하단 힌트에 "나머지 승인" / "나머지 폐기" — 몇 개 처리한 후 한 클릭으로 배치 마무리. "요약으로"는 C-3로 돌아가되 이미 결정한 사실들 진행 상태 유지.

Graph preview — see how the fact fits

Graph preview — 사실이 어떻게 들어가는지 보기

Optional expansion. User clicks "Show graph preview" to see — in plain text — what nodes will be created and what existing objects this fact connects to. No diagram, no learning curve. Three sections:

선택 확장. 사용자가 "그래프 미리보기" 클릭하면 평문으로 — 어떤 노드가 생기고 어떤 기존 객체와 연결되는지 확인. 다이어그램 없음, 학습 곡선 없음. 세 섹션:

  • Creates — New nodes that will be added (+ marker)
  • Connects to existing — Existing graph objects that will gain a new connection (◐ marker)
  • Related in your graph — Existing facts in similar domain, fn-IDs clickable
  • 새로 생성 — 추가될 새 노드 (+ 마커)
  • 기존 노드와 연결 — 새 연결을 받게 될 기존 객체 (◐ 마커)
  • 그래프 내 관련 사실 — 유사 도메인 기존 사실, fn-ID 클릭 가능

This is hidden by default. Users who don't care can ignore it. Users curious about graph structure can expand and learn the system through use.

기본 숨김. 관심 없는 사용자는 무시 가능. 그래프 구조 궁금한 사용자는 펼쳐서 사용하며 시스템 학습.
